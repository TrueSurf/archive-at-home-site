Features

系统特性

围绕「不浪费任何一点配额」设计的解析网络。

志愿者节点网络

解析消耗节点自己的 E-Hentai 账号配额。网络的整体能力取决于在线节点的数量与配额。

缓存与合并

每个用户独立的 7 天缓存；同一画廊的并发请求自动合并为一次解析。

GP 成本追踪

每次新解析返回实际 GP 消耗并写入任务日志，节点侧可查询统计。

令牌桶限流

令牌以 1 GP/秒自动补充，容量上限约 7 天积累量；不足时稍后重试即可。

Telegram 登录

Telegram 授权即自动注册，登录后分配 API Key，无需邮箱密码。

开放 API

REST API 开放，CORS 不限来源，凭 API Key 即可接入自有客户端。

Getting started

使用流程

三步即可开始解析。

  1. 获取 API Key

    在使用指南进行 Telegram 登录（首次登录自动注册），或向 @ArchiveAtHome_bot 领取。

  2. 选择客户端

    网页版直接使用，或安装油猴脚本、使用 TG Bot、在 JHenTai 中配置归档机器人。

  3. 解析

    粘贴画廊链接，返回归档下载链接。7 天内重复解析同一画廊命中缓存，不消耗 GP。
